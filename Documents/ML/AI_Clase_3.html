Introduccin

El aprendizaje de mquina (Machine Learning) ha evolucionado desde los primeros intentos de dotar a las mquinas con capacidades de razonamiento en la dcada de 1950 hasta convertirse en una herramienta central en la sociedad actual. Desde sistemas de recomendacin en plataformas digitales hasta diagnsticos mdicos asistidos por inteligencia artificial, el aprendizaje de mquina es hoy parte integral de la innovacin tecnolgica.

Una breve historia del Machine Learning

  • 1950s: Alan Turing propone la idea de que las mquinas pueden "pensar". Aparece el Test de Turing.

  • 1960s-1970s: Primeros algoritmos de aprendizaje supervisado y no supervisado, como perceptrones y clustering.

  • 1980s: Avances en redes neuronales con la retropropagacin.

  • 1990s: Auge de algoritmos como Mquinas de Vectores de Soporte (SVM).

  • 2000s: Incremento en la capacidad de cmputo y acceso a grandes volmenes de datos.

  • 2010s presente: Expansin del Deep Learning con aplicaciones en visin por computador, procesamiento de lenguaje natural y robtica.

Ciclo de una Aplicacin de Machine Learning y su Relacin con Design Thinking

El ciclo de vida del Machine Learning puede integrarse con los 7 pasos del Design Thinking, que permiten centrar el desarrollo en las necesidades del usuario. The 7 Steps of the Design Thinking Process

  • Identificacin del Problema (Empatizar y Definir)

Se comienza con la comprensin profunda del problema y de las necesidades del usuario. Aqu se fusionan los pasos de empatizar (comprender al usuario) y definir (plantear claramente el problema).

Ejemplo: Detectar por qu los clientes abandonan un servicio.

  • Recoleccin de Datos (Idear fuente de informacin)

La recopilacin de datos es equivalente a generar ideas sobre cmo obtener insumos relevantes. Se identifican fuentes de datos confiables (bases de datos, sensores, encuestas, interaccin de usuarios).

Ejemplo: Datos de clientes, registros de compras, patrones de navegacin.

  • Preparacin de Datos (Prototipar versin inicial del dataset)

As como en Design Thinking se prototipan soluciones, en ML se preparan los datos para ser tiles. Incluye limpieza, normalizacin y seleccin de variables.

Ejemplo: Eliminar duplicados, transformar variables categricas en numricas.

  • Ingeniera de Modelos (Prototipar construir el modelo)

Se disean y entrenan los algoritmos con los datos. Aqu se traduce la idea a un prototipo funcional.

Ejemplo: Entrenar un modelo de regresin logstica o una red neuronal para clasificacin.

  • Evaluacin del Modelo (Testear)

Se valida el modelo utilizando mtricas de desempeo y pruebas con datos no vistos, similar a la fase de testeo en Design Thinking.

Ejemplo: Calcular precisin, recall o AUC en un modelo de prediccin de fraudes.

  • Despliegue (Implementar y Testear en entorno real)

El modelo se implementa en un sistema accesible para usuarios. Esta etapa corresponde a la transicin entre testeo y despliegue final en Design Thinking.

Ejemplo: Integrar un sistema de recomendacin en una plataforma de streaming.

  • Mantenimiento y Actualizacin (Iteracin continua)

El ciclo no termina en el despliegue. Los modelos deben actualizarse y mejorarse con nuevos datos, en sintona con la filosofa iterativa de Design Thinking.

Ejemplo: Ajustar un modelo de prediccin de enfermedades con datos mdicos recientes.

Ejemplo: Proyecto - Sistema de Deteccin de Fraude en Tarjetas de Crdito

  • Empatizar / Identificacin del Problema

    Un banco detecta que sus clientes sufren prdidas debido a fraudes con tarjetas. Los usuarios necesitan mayor seguridad y confianza en sus transacciones.

  • Definir

    El objetivo es construir un sistema que identifique transacciones fraudulentas en tiempo real.

  • Idear / Recoleccin de Datos

    Se recopilan datos de transacciones histricas: monto, ubicacin, hora, tipo de compra y si fueron fraude o no.

  • Prototipar / Preparacin de Datos

    Se eliminan duplicados, se manejan valores faltantes y se equilibran las clases (fraude vs. no fraude).

  • Prototipar / Ingeniera de Modelos

    Se prueban modelos como regresin logstica y bosques aleatorios.

  • Testear / Evaluacin del Modelo

    Se selecciona el modelo con mayor recall (capacidad de detectar fraudes).

  • Despliegue

    El sistema se integra en la aplicacin bancaria para monitorear transacciones en tiempo real.

  • Iteracin / Mantenimiento y Actualizacin

    Cada tres meses se reentrena el modelo con nuevas transacciones, ya que los patrones de fraude cambian.

Ejercicio 1: Sistema de Recomendacin de Pelculas

  1. El sistema se actualiza cada vez que se estrenan nuevas pelculas o cambian las preferencias de los usuarios.

  2. Se recopilan datos del historial de visualizacin de los usuarios y sus calificaciones.

  3. Los ingenieros prueban modelos de filtrado colaborativo y redes neuronales para generar recomendaciones.

  4. Los usuarios de la plataforma demandan recomendaciones ms personalizadas.

  5. El modelo seleccionado se prueba en un grupo reducido de usuarios antes del lanzamiento masivo.

  6. Se procesan los datos: se eliminan registros vacos y se normalizan calificaciones.

  7. Finalmente, el sistema se integra en la plataforma y aparece como Pelculas recomendadas para ti.

Ejercicio 2: Diagnstico Mdico Asistido

  1. Se recopilan imgenes mdicas (radiografas y tomografas) de hospitales asociados.

  2. El modelo se despliega en clnicas, ayudando a los doctores a detectar enfermedades pulmonares.

  3. El sistema es evaluado con mtricas de precisin y recall, verificando su utilidad diagnstica.

  4. Se construye un prototipo inicial entrenando una red neuronal convolucional (CNN).

  5. Los mdicos expresan la necesidad de apoyo tecnolgico para diagnsticos ms rpidos y fiables.

  6. Los datos se limpian, se corrigen errores y se segmentan las imgenes para mejorar la calidad.

  7. Cada cierto tiempo, el modelo se reentrena con nuevas imgenes mdicas para mantenerse actualizado.

Ejercicio 3: Optimizacin de Rutas de Entrega

  1. El sistema se prueba con rutas histricas y se comparan los tiempos de entrega antes y despus.

  2. Se procesan los datos eliminando registros incompletos y georreferenciando direcciones.

  3. Se despliega en la aplicacin mvil de los repartidores para sugerir rutas ms rpidas.

  4. El equipo de logstica identifica retrasos en las entregas y busca soluciones tecnolgicas.

  5. Se recopilan datos de trfico, clima y entregas pasadas.

  6. Se desarrollan modelos de prediccin de trfico usando algoritmos de series temporales.

  7. El sistema se actualiza con datos recientes para adaptarse a cambios en patrones de trfico.
