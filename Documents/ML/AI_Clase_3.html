🧠 Machine Learning¶

Introducción¶

El aprendizaje de máquina (Machine Learning) ha evolucionado desde los primeros intentos de dotar a las máquinas con capacidades de razonamiento en la década de 1950 hasta convertirse en una herramienta central en la sociedad actual. Desde sistemas de recomendación en plataformas digitales hasta diagnósticos médicos asistidos por inteligencia artificial, el aprendizaje de máquina es hoy parte integral de la innovación tecnológica.

Una breve historia del Machine Learning¶

  • 1950s: Alan Turing propone la idea de que las máquinas pueden "pensar". Aparece el Test de Turing.

  • 1960s-1970s: Primeros algoritmos de aprendizaje supervisado y no supervisado, como perceptrones y clustering.

  • 1980s: Avances en redes neuronales con la retropropagación.

  • 1990s: Auge de algoritmos como Máquinas de Vectores de Soporte (SVM).

  • 2000s: Incremento en la capacidad de cómputo y acceso a grandes volúmenes de datos.

  • 2010s – presente: Expansión del Deep Learning con aplicaciones en visión por computador, procesamiento de lenguaje natural y robótica.

💾 Ciclo de una Aplicación de Machine Learning y su Relación con Design Thinking¶

El ciclo de vida del Machine Learning puede integrarse con los 7 pasos del Design Thinking, que permiten centrar el desarrollo en las necesidades del usuario. The 7 Steps of the Design Thinking Process

  • Identificación del Problema (Empatizar y Definir)

Se comienza con la comprensión profunda del problema y de las necesidades del usuario. Aquí se fusionan los pasos de empatizar (comprender al usuario) y definir (plantear claramente el problema).

Ejemplo: Detectar por qué los clientes abandonan un servicio.

  • Recolección de Datos (Idear – fuente de información)

La recopilación de datos es equivalente a generar ideas sobre cómo obtener insumos relevantes. Se identifican fuentes de datos confiables (bases de datos, sensores, encuestas, interacción de usuarios).

Ejemplo: Datos de clientes, registros de compras, patrones de navegación.

  • Preparación de Datos (Prototipar – versión inicial del dataset)

Así como en Design Thinking se prototipan soluciones, en ML se preparan los datos para ser útiles. Incluye limpieza, normalización y selección de variables.

Ejemplo: Eliminar duplicados, transformar variables categóricas en numéricas.

  • Ingeniería de Modelos (Prototipar – construir el modelo)

Se diseñan y entrenan los algoritmos con los datos. Aquí se traduce la idea a un prototipo funcional.

Ejemplo: Entrenar un modelo de regresión logística o una red neuronal para clasificación.

  • Evaluación del Modelo (Testear)

Se valida el modelo utilizando métricas de desempeño y pruebas con datos no vistos, similar a la fase de testeo en Design Thinking.

Ejemplo: Calcular precisión, recall o AUC en un modelo de predicción de fraudes.

  • Despliegue (Implementar y Testear en entorno real)

El modelo se implementa en un sistema accesible para usuarios. Esta etapa corresponde a la transición entre testeo y despliegue final en Design Thinking.

Ejemplo: Integrar un sistema de recomendación en una plataforma de streaming.

  • Mantenimiento y Actualización (Iteración continua)

El ciclo no termina en el despliegue. Los modelos deben actualizarse y mejorarse con nuevos datos, en sintonía con la filosofía iterativa de Design Thinking.

Ejemplo: Ajustar un modelo de predicción de enfermedades con datos médicos recientes.

🦾 Ejemplo: Proyecto - Sistema de Detección de Fraude en Tarjetas de Crédito¶

  • Empatizar / Identificación del Problema

    Un banco detecta que sus clientes sufren pérdidas debido a fraudes con tarjetas. Los usuarios necesitan mayor seguridad y confianza en sus transacciones.

  • Definir

    El objetivo es construir un sistema que identifique transacciones fraudulentas en tiempo real.

  • Idear / Recolección de Datos

    Se recopilan datos de transacciones históricas: monto, ubicación, hora, tipo de compra y si fueron fraude o no.

  • Prototipar / Preparación de Datos

    Se eliminan duplicados, se manejan valores faltantes y se equilibran las clases (fraude vs. no fraude).

  • Prototipar / Ingeniería de Modelos

    Se prueban modelos como regresión logística y bosques aleatorios.

  • Testear / Evaluación del Modelo

    Se selecciona el modelo con mayor recall (capacidad de detectar fraudes).

  • Despliegue

    El sistema se integra en la aplicación bancaria para monitorear transacciones en tiempo real.

  • Iteración / Mantenimiento y Actualización

    Cada tres meses se reentrena el modelo con nuevas transacciones, ya que los patrones de fraude cambian.

Instrucciones:

  • Reunirse en equipos
  • Los equipos deben ordenar cada historia siguiendo el ciclo de vida de ML + Design Thinking

Ejercicio 1: Sistema de Recomendación de Películas

  1. El sistema se actualiza cada vez que se estrenan nuevas películas o cambian las preferencias de los usuarios.

  2. Se recopilan datos del historial de visualización de los usuarios y sus calificaciones.

  3. Los ingenieros prueban modelos de filtrado colaborativo y redes neuronales para generar recomendaciones.

  4. Los usuarios de la plataforma demandan recomendaciones más personalizadas.

  5. El modelo seleccionado se prueba en un grupo reducido de usuarios antes del lanzamiento masivo.

  6. Se procesan los datos: se eliminan registros vacíos y se normalizan calificaciones.

  7. Finalmente, el sistema se integra en la plataforma y aparece como “Películas recomendadas para ti”.

Ejercicio 2: Diagnóstico Médico Asistido

  1. Se recopilan imágenes médicas (radiografías y tomografías) de hospitales asociados.

  2. El modelo se despliega en clínicas, ayudando a los doctores a detectar enfermedades pulmonares.

  3. El sistema es evaluado con métricas de precisión y recall, verificando su utilidad diagnóstica.

  4. Se construye un prototipo inicial entrenando una red neuronal convolucional (CNN).

  5. Los médicos expresan la necesidad de apoyo tecnológico para diagnósticos más rápidos y fiables.

  6. Los datos se limpian, se corrigen errores y se segmentan las imágenes para mejorar la calidad.

  7. Cada cierto tiempo, el modelo se reentrena con nuevas imágenes médicas para mantenerse actualizado.

Ejercicio 3: Optimización de Rutas de Entrega

  1. El sistema se prueba con rutas históricas y se comparan los tiempos de entrega antes y después.

  2. Se procesan los datos eliminando registros incompletos y georreferenciando direcciones.

  3. Se despliega en la aplicación móvil de los repartidores para sugerir rutas más rápidas.

  4. El equipo de logística identifica retrasos en las entregas y busca soluciones tecnológicas.

  5. Se recopilan datos de tráfico, clima y entregas pasadas.

  6. Se desarrollan modelos de predicción de tráfico usando algoritmos de series temporales.

  7. El sistema se actualiza con datos recientes para adaptarse a cambios en patrones de tráfico.

Descripción de sistemas de Machine Learning reales¶

Para aprender a diseñar sistemas de aprendizaje automático, resulta útil leer casos prácticos para ver cómo equipos reales gestionan diferentes requisitos y limitaciones de implementación. Muchas empresas —Airbnb, Lyft, Uber y Netflix, por nombrar algunas— publican excelentes blogs tecnológicos donde comparten su experiencia con el aprendizaje automático para mejorar sus productos o procesos.

Instrucciones:

  • Cada equipo escojera uno de los casos prácticos relacionados y hará una presentación al grupo.
