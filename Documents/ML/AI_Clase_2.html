Machine Learning

Como subcampo de la IA, el Machine Learning se centra en la creacin de algoritmos y modelos estadsticos que permiten a las computadoras mejorar su rendimiento en tareas especficas mediante el aprendizaje de los datos.

A diferencia de la programacin tradicional, los sistemas de aprendizaje automtico adquieren conocimiento de los datos, detectan patrones y toman decisiones o predicciones.

El aprendizaje automtico ha cobrado una enorme importancia en diversos sectores, como la salud, las finanzas, el marketing y la tecnologa. Ha revolucionado la toma de decisiones, la automatizacin de procesos y la extraccin de informacin de grandes conjuntos de datos.

Sus principales beneficios incluyen la automatizacin, el anlisis predictivo, la personalizacin, el diagnstico mdico y el procesamiento del lenguaje natural.

Tipos de Aprendizaje en ML

En Machine Learning existen diferentes formas en que una mquina puede aprender a partir de los datos. Los tres tipos principales son: aprendizaje supervisado, no supervisado y por refuerzo.

Aprendizaje Supervisado - Supervise Learning

En este enfoque, el modelo aprende a partir de un conjunto de datos que incluye entradas y salidas conocidas. Es decir, el sistema recibe ejemplos con las respuestas correctas y ajusta sus predicciones para acercarse a esas respuestas.

  • Un modelo que aprende a clasificar correos electrnicos como spam o no spam, usando ejemplos previamente etiquetados.

  • Reconocimiento de voz: Un sistema que aprende a transcribir audio a texto, entrenado con grabaciones de voz y sus transcripciones correctas.

  • Prediccin del clima: Un modelo que, con datos histricos (temperatura, humedad, presin), predice si maana llover o no.

Algunos de los importantes algoritmos de aprendizaje supervizado son:

  • Linear Regression
  • Logistic Regression
  • Decision Trees
  • Random Forest
  • Support Vector Machines (SVM)
  • Naive Bayes
  • Gradient Boosting Machines
  • Adaptive Learning Algorithms

Aprendizaje No Supervisado - Unsupervised learning

Aqu, los datos no tienen etiquetas; el objetivo es encontrar patrones, estructuras o relaciones ocultas. El modelo organiza o agrupa la informacin por s mismo.

  • Un algoritmo que agrupa clientes en un supermercado segn sus hbitos de compra, sin que exista una categora predefinida.

  • Segmentacin de mercado: Una empresa usa datos de consumo para agrupar clientes en perfiles (ejemplo: compradores frecuentes, ocasionales, de lujo), sin etiquetas predefinidas.

  • Compresin de imgenes: Algoritmos que reducen el tamao de una imagen encontrando patrones de redundancia sin necesidad de ejemplos previos.

Algunos de los ms importantes algoritmos de aprendizaje no supervisado son:

  • Clustering
    • Clustering Algorithms
    • K-Means
    • DBSCAN
    • Hierarchical Cluster Analysis (HCA)
  • Anomaly detection and novelty detection
    • One-class SVM
    • Isolation Forest
    • Hidden Markov Models (HMM)
    • K- Nearest Neighbors (KNN)
  • Visualization and dimensionality reduction
    • Principal Component Analysis (PCA)
    • Kernel PCA
    • Locally Linear Embedding (LLE)
    • t-Distributed Stochastic Neighbor Embedding (t-SNE)
  • Association rule learning
    • Apriori
    • Eclat

Aprendizaje por Refuerzo - Reinforcement Learning

En este caso, el modelo aprende mediante prueba y error, recibiendo recompensas o penalizaciones por las acciones que realiza en un entorno. Su meta es encontrar la mejor estrategia para maximizar la recompensa.

  • Un robot que aprende a caminar recibiendo puntos positivos cuando avanza y negativos cuando se cae.

  • Videojuegos: Un agente que aprende a jugar a Pac-Man tomando decisiones en tiempo real, recibiendo puntos por comer frutas y evitando perder vidas.

  • Robtica industrial: Un brazo robtico que aprende a ensamblar piezas mediante prueba y error, optimizando movimientos para reducir errores y maximizar eficiencia.

Principales desafos del Machine Learning

En resumen, dado la tarea principal es seleccionar un algoritmo de aprendizaje y entrenarlo con datos, existen varios factores que pueden hacer fallar este proceso los cuales son los datos incorrectos o un algoritmo incorrecto.

Problemas en los datos

  • Cantidad insuficiente de datos de entrenamiento: El aprendizaje automtico an no est del todo desarrollado; se necesitan muchos datos para que la mayora de sus algoritmos funcionen correctamente. Incluso para problemas muy sencillos, se suelen necesitar miles de ejemplos, y para problemas complejos como el reconocimiento de imgenes o de voz, pueden necesitarse millones de ejemplos.

Tomado de: Banko, Michele and Brill, Eric. Scaling to Very Very Large Corpora for Natural Language Disambiguation, 2001 (https://dl.acm.org/doi/10.3115/1073012.1073017)

  • Datos de entrenamiento no representativos: Para generalizar correctamente, es fundamental que los datos de entrenamiento sean representativos de los nuevos casos a los que se desea generalizar. Esto se aplica tanto al aprendizaje basado en instancias como al aprendizaje basado en modelos.

Tomado de: Gron, Aurlien. Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow

Al utilizar un conjunto de entrenamiento no representativo, entrenamos un modelo que probablemente no realice predicciones precisas, especialmente para pases muy pobres y muy ricos. Es crucial utilizar un conjunto de entrenamiento que sea representativo de los casos a los que se desea generalizar. Esto suele ser ms difcil de lo que parece: si la muestra es demasiado pequea, se obtendr ruido de muestreo (es decir, datos no representativos debido al azar), pero incluso muestras muy grandes pueden no ser representativas si el mtodo de muestreo es defectuoso. Esto se llama sesgo de muestreo.

  • Datos de baja calidad: datos de entrenamiento llenos de errores, valores atpicos y ruido (por ejemplo, debido a mediciones de baja calidad).

  • Datos con Caractersticas Irrelevantes: Como dice el refrn: basura entra, basura sale. El sistema solo podr aprender si los datos de entrenamiento contienen suficientes caractersticas relevantes y no demasiadas irrelevantes.

Problemas en los Algoritmos

  • Sobreajuste de los datos de entrenamiento: significa que el modelo funciona bien con los datos de entrenamiento, pero no generaliza bien.

  • Subajuste de los datos de entrenamiento: ocurre cuando el modelo es demasiado simple para comprender la estructura subyacente de los datos. las principales opciones para solucionar este problema:

    • Seleccionar un modelo ms potente, con ms parmetros.
    • Incorporar mejores caractersticas al algoritmo de aprendizaje (ingeniera de caractersticas).
    • Reducir las restricciones del modelo.

Problemas de hardware y software

Los departamentos de TI se enfrentan a retos de hardware y software a la hora de respaldar el entrenamiento de modelos de ML. Entre los posibles obstculos se encuentran disponer de suficiente potencia informtica y capacidad de almacenamiento, recursos de datos y herramientas de compatibilidad e integracin para llevar un proyecto hasta su finalizacin.

Problemas en la contratacin de personal calificado

Para desarrollar, gestionar e iterar el entrenamiento de modelos de ML se necesitan personas con conocimientos especializados en distintas disciplinas tcnicas.

  • Demanda de talentos
  • Falta de profesionales capacitados en IA

Problemas en la gestin de proyectos de ML

Los proyectos de IA empresarial pueden ser costosos y requerir muchos recursos. Ms all de las preocupaciones inmediatas de desarrollo de modelos, seleccin de fuentes de datos y entrenamiento de modelos de IA, la gestin requiere un delicado equilibrio de supervisin financiera, tecnolgica y de programacin.

  • Fallas de comunicacin
  • Expectativas desalineadas

En Resumen

  • El Machine Learning consiste en lograr que las mquinas mejoren en alguna tarea aprendiendo de los datos, en lugar de tener que codificar reglas explcitamente.
  • Existen muchos tipos diferentes de sistemas de aprendizaje automtico: supervisados o no, por lotes o en lnea, basados en instancias o en modelos.
  • En un proyecto de aprendizaje automtico, se recopilan datos en un conjunto de entrenamiento y se alimenta este conjunto a un algoritmo de aprendizaje.
    • Si el algoritmo se basa en modelos, ajusta algunos parmetros para ajustar el modelo al conjunto de entrenamiento (es decir, para realizar buenas predicciones sobre el propio conjunto de entrenamiento) y, con suerte, tambin podr realizar buenas predicciones en nuevos casos.
    • Si el algoritmo se basa en instancias, simplemente aprende los ejemplos de memoria y los generaliza a nuevas instancias utilizando una medida de similitud para compararlas con las instancias aprendidas.
  • El sistema no funcionar bien si el conjunto de entrenamiento es demasiado pequeo, si los datos no son representativos, si presentan ruido o si estn contaminados con caractersticas irrelevantes (basura entrante, basura saliente).
  • Por ltimo, el modelo no debe ser ni demasiado simple (en cuyo caso se subajustar) ni demasiado complejo (en cuyo caso se sobreajustar).
